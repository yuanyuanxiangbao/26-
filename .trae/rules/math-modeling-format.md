# 数学建模项目文件夹组织规范

## 适用范围
本规范适用于26电工杯数学建模竞赛项目，所有代码编写和文件组织必须遵循此规范。

## 文件夹结构

### 基本结构
```
项目根目录/
├── 问题1/
│   ├── 1.1/
│   │   ├── code/          # 代码文件
│   │   ├── results/       # 生成的结果（图表、输出文件）
│   │   ├── data/          # 生成的中间数据
│   │   └── conclusion.txt # 结论文件
│   ├── 1.2/
│   │   ├── code/
│   │   ├── results/
│   │   ├── data/
│   │   └── conclusion.txt
│   └── 1.3/
│       ├── code/
│       ├── results/
│       ├── data/
│       └── conclusion.txt
├── 问题2/
│   ├── 2.1/
│   │   ├── code/
│   │   ├── results/
│   │   ├── data/
│   │   └── conclusion.txt
│   └── 2.2/
│       ├── code/
│       ├── results/
│       ├── data/
│       └── conclusion.txt
└── ...
```

## 命名规范

### 父文件夹
使用中文"问题+数字"，如`问题1`、`问题2`、`问题3`

### 子文件夹
使用问题编号，如`1.1`、`1.2`、`2.1`、`2.2`

### 功能子文件夹
- `code/`：存放所有.py代码文件
- `results/`：存放生成的图表、分析报告、可视化结果
- `data/`：存放处理后的中间数据、CSV文件、预测结果
- `conclusion.txt`：存放该子问题的结论总结

## 代码编写规范

### 路径配置
每个代码文件开头必须配置路径：
```python
# 路径配置
BASE_DIR = "问题1/1.1"
DATA_DIR = f"{BASE_DIR}/data"
RESULTS_DIR = f"{BASE_DIR}/results"
```

### 结果保存
- 数据结果保存到`DATA_DIR`
- 图表结果保存到`RESULTS_DIR`
- 文件名使用英文，避免编码问题

### 独立性
每个子问题的代码完全独立，不依赖其他子问题的代码文件

## 结论文件规范

每个子问题完成后，在对应的`conclusion.txt`中记录：
- 使用的模型/方法
- 关键参数
- 主要结果
- 结论总结

格式：
```
问题：1.1 XXXX问题
方法：XXX模型/算法
关键参数：param1=value1, param2=value2
主要结果：
- 结果1
- 结果2
结论：XXX
```

## 强制要求

1. 所有新代码必须按照此结构存放
2. 禁止在根目录直接存放代码文件
3. 禁止不同子问题共用同一个代码文件
4. 路径使用相对路径，确保可移植性
5. 文件名使用英文，避免Windows编码问题
6. 定期清理临时文件，保持目录整洁
